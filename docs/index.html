BranchClaw Docs

Git-first agent workspace for long-lived software iteration

BranchClaw centers one repository around a long-lived run, planner-owned feature backlog, isolated Git worktrees, reviewable batches, and a separate integration branch before promotion to main.

uv run branchclaw daemon start

Product Loop

Pick a workdir, open a run, let the planner maintain feature backlog, review the worktree graph, approve batch merge to integration_ref, then promote separately.

Human-in-the-Loop

If you want every important gate to stay manual, use the dedicated Three.js E2E runbook and keep plan, archive, merge-to-integration, and promote under explicit approval.
